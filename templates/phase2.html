2-EDIT Edit the record by loading its current values into the form

To accomplish this, I decided to create a couple of separate pages, list.html and edit-record.html based on the buggy-form. The reason for this is because, I wanted to allow a user to select from a list of buggies available and then edit them using a separate form. This task involved writing scripts jQselector.js, loadselected.js and submit.js.

The function display_buggies() in app.py reads all buggies from the database table and displays them in list.html. Once a user selects a buggy, the selection is identified by a jQuery and stored in a hidden form which is submitted to Flask via submit.js. User selection data is also passed as JSON from Flask to edit-record.

Once a selection is made, the edit-record.html is rendered for the user to edit.

This task is tested to be working well.

2-COST Calculate and save the game cost of the buggy

To calculate the cost, I though of storing all the required data in separate tables 'Tyre Type' and 'Offensive Capability'). I then wrote functions to calculate the cost of the buggy by reading these tables for the cost parameters calculating the total cost in the function calculate_cost() in app.py. Please note, I have renamed the buggies table to Buggy. Cost calculation is verified to be working correctly.

This task is tested to be working well.

2-RULES Add validation according to the game rules

This involved writing several if statements in create_buggy() to check the form data according to the rules of the game. I modifies the validate_integer() function to accept two parameters in order to check for upperbound value as well as for integer type. These tests are documented in the TESTING.md file and are all verified to be working correctly.

This task is tested to be working well.
